# Codex Banner Studio

Codex Banner Studio は、Codex を使って広告バナー候補を生成・確認・保存・整理するためのローカルWebアプリです。

商品情報と参考画像を登録し、複数のバナー案をまとめて生成し、良い候補を選んで仕上げ、ローカルのライブラリに保存して管理する用途を想定しています。

## 特徴

このアプリは、OpenAI APIキーを直接使うのではなく、ローカルの Codex CLI / App Server 経由でCodexに処理を依頼します。

そのため、Codexが利用できるChatGPTサブスクリプションでログインしていれば、広告訴求案の作成から画像生成まで、Codexのレートリミット内で利用できます。API利用量の管理よりも、ChatGPT / Codex の利用枠を前提にしたローカル補助ツールとして試しやすい構成です。

## できること

- ブランド、商品名、価格情報、メモ、参考画像を商品マスタとして登録
- 商品情報からCodexに広告訴求案・バナー案を作成させる
- 複数候補を1枚に並べた分割シート形式でバナー画像を生成
- 分割シートから個別候補を選択して仕上げ生成
- 生成したバナーをフォルダ型のライブラリに保存
- 保存済みバナーの検索、評価、名前変更、整理、ダウンロード
- Codex接続や生成処理のログ確認

## 仕組み

アプリ本体は Next.js 製のローカルWebアプリです。

画像生成を含む生成処理は、ローカル環境の Codex CLI / App Server 経由でCodexに依頼します。アプリは商品マスタ、参考画像の説明、ユーザー指定、生成設定をもとにプロンプトを組み立て、同じPC上のCodexへリクエストします。

生成画像、商品マスタ、ライブラリ画像、実行ログはローカルPC内に保存されます。ホストされたバックエンドや共有データベースを使わず、手元で試しやすい構成です。

## 起動方法

### Mac

`start-mac.command` をダブルクリックします。

初回起動時は依存パッケージのセットアップが走るため少し時間がかかります。起動後、ブラウザが自動で開きます。

### Windows

`start-windows.bat` をダブルクリックします。

起動時は専用の黒い画面が開きます。エラーが出ても画面が閉じないようにしているため、表示内容を確認できます。起動ログは以下にも保存されます。

```text
public\data\startup-log.txt
```

Node.js が入っていない場合は、起動ファイルが Node.js の公式ダウンロードページを開きます。LTS版をインストールしてから、もう一度 `start-windows.bat` を実行してください。

## 必要なもの

画像生成まで使うには、実行するPCに以下が必要です。

- Node.js LTS
- Codex CLI
- Codex CLI で利用できるログイン済みの Codex / ChatGPT アカウント

Codex が未設定でもアプリ画面自体は開けますが、画像生成には Codex CLI のセットアップとログインが必要です。アプリ内の接続テストで状態を確認できます。

## 手動起動

```bash
npm install
npm run dev -- --hostname 127.0.0.1 --port 3000
```

ブラウザで以下を開きます。

```text
http://127.0.0.1:3000
```

## ローカルデータ

生成データやユーザーが登録したデータは、主に `public` 配下に保存されます。

- 生成シート: `public/generated`
- ライブラリ保存画像: `public/saved-banners`
- 商品画像: `public/products`
- 商品マスタ: `public/data/products.json`
- ブランドマスタ: `public/data/brands.json`
- リクエストログ: `public/data/request-log.jsonl`
- 起動ログ: `public/data/startup-log.txt`

これらはローカル実行時のデータとして扱い、Git管理から外しています。

空のサンプルマスタとして以下を含めています。

- `public/data/products.example.json`
- `public/data/brands.example.json`

## Git管理しないもの

以下は `.gitignore` で除外しています。

- 商品マスタ
- ブランドマスタ
- 商品画像
- 生成画像
- ライブラリ保存画像
- リクエストログ
- 起動ログ
- 依存パッケージとビルド成果物

これにより、商品データや生成物を含めずにプロジェクト本体だけを共有できます。

## 配布時の注意

Windowsでも扱いやすいよう、起動ファイル名は英数字にしています。

- Mac: `start-mac.command`
- Windows: `start-windows.bat`

別PCへ渡す場合は、`node_modules`、`.next`、生成画像、保存画像、商品画像、ログを含めない形で配布する想定です。初回起動時に依存パッケージがインストールされます。

## うまく動かないとき

- アプリが開かない場合は、別プロセスが `3000` 番ポートを使っていないか確認してください。
- 画像生成だけ失敗する場合は、アプリ内の Codex 接続テストを確認してください。
- Windowsで起動に失敗する場合は、黒い起動画面と `public\data\startup-log.txt` を確認してください。
- Codex が見つからない場合は、通常のターミナルで `codex` コマンドが動くか確認してください。
